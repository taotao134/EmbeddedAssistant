# 嵌入式调试台使用说明

## 启动

只运行：

```text
ReleasePackages\嵌入式调试台-win-x64\DeviceDebugStudio.exe
```

程序为 Windows x64 自包含版本，不需要安装 .NET 运行库。

## 设备工作区

- 左侧选择或新建设备；连接参数和快捷命令会自动保存，也可点击“保存设备配置”立即保存并查看保存位置。
- “配置操作”可选择一个或多个 `sscom51.ini`、`netassist.cfg` 文件导入，不会修改原文件。
- “配置操作”还支持设备 JSON 导入导出、打开捕获 `.db`，以及修改设备配置保存目录。
- 默认配置目录为 `%LocalAppData%\DeviceDebugStudio\Profiles`，修改后会保存在本机设置中。

## 通信

- 串口支持自动热插拔刷新、自定义波特率、DTR、RTS 和自动重连。
- TCP 服务器向空目标发送时会广播给全部已连接客户端。
- UDP 支持单播、广播和组播；目标格式为地址与端口。
- 经典蓝牙串口通过 Windows 配对后作为 COM 口使用。
- BLE GATT 先扫描设备，再浏览服务与特征值；选择树节点可自动填入读、写或通知 UUID。

## 数据与协议

- 终端支持 ASCII、UTF-8、GBK 和 HEX，清空界面不会删除捕获文件。
- 终端“表格”页用于按字段查看；“连续文本”页支持像文本编辑器一样跨行拖选，并可用 `Ctrl+C` 或右键复制。
- 每次连接都会在 `%USERPROFILE%\Documents\DeviceDebugStudio\Captures` 创建 SQLite 捕获文件。
- 帧解析支持原始块、文本行、分隔符、固定长度、长度字段和空闲间隔。
- 帧字段、字节序、比例、位域、转义和校验在“帧解析”页的 JSON 模板中配置。
- Modbus 页支持常用 RTU/TCP 请求；启用从站模拟后，串口按 RTU 响应，TCP Server 按 Modbus TCP 响应。

## 快捷命令

- 命令可使用文本或 HEX，可选择换行、校验和循环间隔。
- 顶部搜索框按名称或发送内容定位命令，排序可选使用频率、最近使用或手动顺序。
- “命令变量（可选）”按 `key=value` 每行填写，在命令中使用 `${key}` 引用；例如填写 `id=01` 后，`${id}` 会替换为 `01`。
- 连接参数在会话建立后会锁定；断开后才能切换端口或网络端点。

## 在线更新

在“设置”窗口的“在线更新”区域填写 GitHub 仓库（例如 `owner/repository`），可选择启动时检查。发布版本的 GitHub Release 标签应为 `v1.2.3` 形式，并上传 `win-x64` ZIP 和对应的 `.sha256` 校验文件。程序会在后台检查新版本，下载后校验 SHA-256，再由安装代理覆盖程序文件并重启；设备配置和捕获文件保存在用户数据目录，不会被更新包覆盖。
